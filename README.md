# Personal Intelligence OS

This is a personal tool you can use to observe your own workflows and build automation.

## What it does

- Scans local shell history files (zsh/bash/fish) for repeated command patterns.
- Optionally ingests local ActivityWatch data (if ActivityWatch server is running on `127.0.0.1:5600`).
- Ingests optional low-overhead intent events (`/api/intent/events`) from adapters (browser/API hooks).
- Detects workflows and shows:
  - Workflow details
  - Frequency
  - Tools used
  - Category (`research`, `gtm`, `ops`, `engineering`, `analytics`, `admin`)
- Proposes automation ideas:
  - Uses local Ollama if available
  - Falls back to deterministic heuristics if Ollama is unavailable

### Intent API

- `GET /api/intent/schema` returns expected payload format.
- `POST /api/intent/events` accepts either:
  - a single event object, or
  - `{ "events": [ ... ] }`, or
  - a raw JSON array of events.

Example:

```bash
curl -sS -X POST http://127.0.0.1:5180/api/intent/events \
  -H 'Content-Type: application/json' \
  -d '{
    "timestamp":"2026-03-09T18:41:00Z",
    "source":"manual_test",
    "tool":"browser",
    "domain":"linkedin.com",
    "intent":"job_search",
    "action":"open_job",
    "stage":"exploration",
    "confidence":0.9
  }'
```

Stored file path (default): `~/.personal_intelligence_os/intent_events.ndjson`

Optional env override:

```bash
export WORKFLOW_INTENT_EVENT_STORE="$HOME/.personal_intelligence_os/intent_events.ndjson"
```

## Optional: local LLM ideas via Ollama

If Ollama is running, ideas are generated by local model first.

```bash
# Single model (Qwen preferred)
export WORKFLOW_LLM_MODEL="qwen2.5:7b-instruct"

# Optional fallback chain (tries in order)
export WORKFLOW_LLM_MODELS="qwen2.5:7b-instruct,llama3.2:3b"
export OLLAMA_HOST="http://127.0.0.1:11434"
./run_dashboard.sh
```
